# JavaScript 函數與陣列實作挑戰

這份文件記錄了 10 道 JavaScript 進階函數、Callback 與陣列操作題目的解題思路與實用訣竅。

## 1. Callback 基礎實作
* **解題步驟**：設計一個接收三個參數的函數，其中第三個參數 `action` 預期是一個函數。在函數內部直接 `return action(num1, num2)` 即可。呼叫時傳入定義好的匿名函數。
* **訣竅**：將函數當作變數一樣傳遞是 JS 的核心特性。傳入 Callback 時記得「不要加括號」，因為你是在傳遞函數本體，而不是傳遞執行結果。

## 2. 匿名函數與立即執行 (IIFE)
* **解題步驟**：使用 `(function() { ... })();` 的語法將程式碼包裝起來。
* **訣竅**：IIFE (Immediately Invoked Function Expression) 常用來建立獨立的作用域（Scope），避免內部的變數（如 `count`）污染全域環境。

## 3. 箭頭函數與陣列轉換
* **解題步驟**：呼叫 `prices.map()` 並在裡面放入箭頭函數 `price => price * 0.8`。
* **訣竅**：箭頭函數如果在單行內直接回傳結果，可以省略 `{}` 和 `return` 關鍵字，這讓程式碼看起來極度簡潔。

## 4. 陣列參數的「破壞性修改」
* **解題步驟**：使用 `arr.pop()` 移除最後一個元素，再用 `arr.unshift("Start")` 將元素加到最前面。
* **訣竅**：在 JS 中，陣列與物件都是「傳址（Pass by Reference/Sharing）」。傳入函數內的陣列其實是指向同一個記憶體位址，因此直接使用修改原陣列的方法（如 `pop`, `push`, `shift`, `unshift`）會直接改變外部的資料。

## 5. 函數回傳函數 (Higher-Order Function)
* **解題步驟**：外層函數接收 `factor`，並在內部 `return` 一個箭頭函數 `(n) => n * factor`。
* **訣竅**：這裡利用了「閉包 (Closure)」的特性。被回傳的內部函數會「記住」外層函數的變數 `factor`，即使外層函數已經執行完畢，內層函數依然可以使用該變數。

## 6. Callback 篩選器
* **解題步驟**：建立一個空陣列，使用 `for` 迴圈遍歷傳入的陣列。將陣列當前元素丟入 `callback(item)` 中，如果結果為真（Truthy），就把元素 `push` 到新陣列中。
* **訣竅**：這就是在模擬原生 `Array.prototype.filter` 的底層行為。理解這題能幫你徹底搞懂所有陣列迭代方法（map, filter, reduce）是如何運作的。

## 7. 箭頭函數處理物件
* **解題步驟**：使用 `users.filter(user => user.age >= 18)`。
* **訣竅**：當陣列裡面包著物件時，Callback 參數拿到的就會是該物件（這裡命名為 `user`），接著直接用點記號 `user.age` 來取值做邏輯判斷即可。

## 8. 參數傳址陷阱：重新賦值 vs 修改
* **解答**：`listA` 變成 `[1, 2, 99]`，而 `listB` 依然是 `[3, 4]`。
* **訣竅**：這題是非常經典的面試題。傳入函數的 `a` 和 `b` 其實是外部陣列的「記憶體位址拷貝」。
  - 當你對 `a` 執行 `.push(99)`，你是順著位址去修改原本的記憶體內容。
  - 當你寫 `b = [100]` 時，你是將 `b` 這個變數的指標「切斷」，重新指向一個全新的陣列 `[100]`，這並不會影響外部 `listB` 原本指向的記憶體位置。

## 9. 延遲執行的 Callback
* **解題步驟**：使用 `setTimeout`，第一個參數放箭頭函數，內部宣告陣列並用 `.join(" ")` 組合字串印出；第二個參數設定為 `2000`（毫秒）。
* **訣竅**：`setTimeout` 是非同步（Asynchronous）操作的基礎。學會把邏輯包在 Callback 中交給 `setTimeout` 是進入 JS 異步程式設計的第一步。

## 10. 綜合應用：計算總價
* **解題步驟**：先用 `cart.reduce((acc, curr) => acc + curr, 0)` 將陣列內數字加總得出 `sum`。接著將 `sum` 當作參數傳給 `discountFunc(sum)` 並回傳。
* **訣竅**：`reduce` 是一個強大的累加器。這題展示了如何將「資料處理 (加總)」與「商業邏輯 (自訂折扣規則)」拆分。把折扣規則用 Callback 傳入，能讓這支函數具備極高的重用性（例如可以傳入打折的 Callback，或是折抵固定金額的 Callback）。
